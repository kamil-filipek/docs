# What are the simplest use cases to get started with CodeMie?

The most simple use cases to start working with CodeMie are the following.  
Related to code:

- Talk to your Code.
- Newcomer training
- Code Review

To start working with code you just need to configure your git token in integration. Then add your Git Repository using “Add new Git Repository”. After that, you can create your assistant based on “[Template] Project Onboarding Assistant” with an indexed repository as Datasource Context and start chatting.

You can also use CodeMie for non-code scenarios like:

- User Story Creation. You need to configure your Jira in the token in integration, after that you can create an assistant based on the template “[Template] Epic/User story Composer” and start working with your Jira. You can read, create, update issues, change statuses, log time and add comments.
- Research Summary and Reporting from Web Sources. Please create an assistant based on the template “[Template] Google Search Assistant” and get new experience from searching and summarizing the information
- FAQ helper. This scenario will require more effort from your side. You will need to create a Google Document as described in CodeMie documentation, add it via Add new Data Source and create an assistant based on the template “[Template] Project Onboarding Assistant” with a Datasource Context set to create a Data Source.

Should you have any specific questions do not hesitate to ask me!
