# How does A2A handle multi-turn conversations?

A2A Protocol supports multi-turn conversations with conversational flows and state management. When working with A2A remote assistants in CodeMie:

- You can type questions and requests
- Continue conversations with context preserved
- The conversation works just like with regular CodeMie assistants
- Each remote assistant maintains its own conversation context

The Currency Conversion Agent example demonstrates multi-turn conversations with follow-up questions and conversational memory across interactions.
