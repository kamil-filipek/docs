# How to start working with my Confluence? How to integrate Datasource Context documentation with CodeMie? integrations with KB? integrations with Confluence?

To integrate Datasource Context documentation with CodeMie, follow the steps below:

1. In Data Sources generate personal access token:

- Click on your profile icon in the right upper corner.
- Click by Settings.
- Click by Personal Access Tokens.
- Click Create token.
- Fill Token Name filed.
- Click Create.

2. In the CodeMie main menu, click the integrations button.
3. Select Integration Type: User or Project and click Create
4. Select the Project Name.
5. Select the Credential Type: Confluence.
6. Fill in the Alias is a representation of the user setting.
7. Fill in the URL field. For example: [https://kb.example.com/](https://jiraeu.epam.com/) .
8. Fill in the Token field with the token created at step 1.
9. Click + Save.
10. Click Explore Assistant, select Templates and choose for example [Template] Confluence Assistant.
11. Select your Project, Name and select Alias name Project Management - Generic Confluence from the drop down list.
12. Click Create.
13. Click Explore Assistant, select My Assistant and choose by Name your assistant.  
    Note: Tokens have an expiration date.  
    Note: Please be aware that an alternative method exists to access the "Add new Data Source" interface. While adding or modifying your assistant, you have the option to click the "Add Data Source" button. Additionally, if your CodeMie project currently has no Data Sources connected, a link will be provided that directly takes you to the "Add new Data Source" screen. Keep in mind that this link is only visible in the absence of any added Data Sources.

Note: Confluence page processing has a limit of 1000 pages.
